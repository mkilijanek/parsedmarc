# Project Working Rules

## Documentation-Driven Delivery

This repository is delivered as **documentation-driven** work.

Required execution order for substantial changes:
1. Define the need, scope, and justification.
2. Document the intended change and acceptance criteria.
3. Implement the change.
4. Update repository documentation to match the implementation.
5. Run tests and fix regressions.
6. Finish with documentation that reflects the final delivered state.

## Change Journal

Maintain `change.log` as the working journal for:
- planned tasks,
- execution notes,
- important commands,
- observations,
- blockers,
- decisions,
- lessons learned.

Before major implementation phases, record:
- what is about to change,
- why it is needed,
- how success will be verified.
