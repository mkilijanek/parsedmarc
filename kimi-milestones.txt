Issue 1.6.1-01: Define adapter contracts for feeds and export targets
-----------------------------------------------------------------------
Status: OPEN
Priority: HIGH
Assignee: TBD
Labels: architecture, adapters, interfaces

Opis:
Zdefiniowanie formalnych interfejsów dla adapterów feedów i eksportu.

Wymagania:
1. Interfejs FeedAdapter (abstract base class):
   ```python
   class FeedAdapter(ABC):
       @abstractmethod
       async def fetch(self, ctx: FetchContext) -> FetchResult:
           ...

       @abstractmethod
       def validate_config(self, config: dict) -> ValidationResult:
           ...

       @abstractmethod
       def get_capabilities(self) -> CapabilityMetadata:
           ...

       @abstractmethod
       async def health_check(self) -> HealthStatus:
           ...
   ```

2. Interfejs ExportAdapter:
   ```python
   class ExportAdapter(ABC):
       @abstractmethod
       def export(self, indicators: Iterator[Indicator], format: str) -> ExportResult:
           ...

       @abstractmethod
       def supports_format(self, format: str) -> bool:
           ...
   ```

3. Modele danych:
   - FetchResult (items, stats, stop_reason, details)
   - FetchContext (since, until, limit, filters)
   - CapabilityMetadata (supported_filters, max_limit, auth_type)
   - HealthStatus (healthy, latency, message)
   - ValidationResult (valid, errors)

4. Protocol definitions (Python 3.8+):
   - Dla duck typing gdy ABC za ciężkie

Kryteria akceptacji:
- [ ] Interfejsy zdefiniowane i udokumentowane
- [ ] Type hints kompletne
- [ ] Protocol classes dla duck typing
- [ ] Testy dla interfejsów (abc meta tests)

Pliki do utworzenia:
- app/adapters/__init__.py
- app/adapters/base.py (interfejsy)

Issue #132: Nginx edge can return 502 after app container recreation
---------------------------------------------------------------------
Priority: HIGH
Status: OPEN

Opis:
Po recreacji app kontenera nginx zwraca 502 (stale upstream).

Wymagania:
1. Root cause fix:
   - Resolver z TTL w nginx.conf
   - Health check dla upstream
   - Keepalive connections

2. Config changes:
   ```nginx
   upstream app {
       server app:8080;
       keepalive 32;
   }

   server {
       location / {
           resolver 127.0.0.11 valid=10s;  # Docker DNS
           proxy_pass http://app;
           proxy_http_version 1.1;
           proxy_set_header Connection "";
       }
   }
   ```

3. Alternative: consul-template lub nginx plus dla dynamic upstreams

Pliki do modyfikacji:
- nginx/templates/default.conf.template
- docker-compose.yml (healthcheck dla app)
